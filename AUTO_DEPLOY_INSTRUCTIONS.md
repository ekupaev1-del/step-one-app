# ✅ Автоматизация деплоя настроена

## Что было исправлено

**Проблема:** Изменения в `dev` не отражались в production, потому что бот использует production URL, который обновляется только из `main`.

**Решение:** Настроен автоматический workflow: после каждого коммита в `dev` → автоматический мерж в `main` → production обновляется.

## Текущий статус

✅ Все изменения из `dev` запушены в `main`  
✅ Production должен обновиться автоматически через Vercel  
✅ Бот использует production URL: `https://step-one-app.vercel.app`

## Проверка

1. Подождите 1-2 минуты (Vercel деплоит production)
2. Откройте в браузере: `https://step-one-app.vercel.app`
3. Проверьте, что видны ваши последние изменения
4. Протестируйте в Telegram боте

## Будущий workflow

Теперь при работе в `dev`:
1. Делаете изменения
2. Коммитите: `git commit -m "..."`  
3. Пушите: `git push origin dev`
4. **Автоматически** мержится в `main` и production обновляется

Или используйте скрипт:
```bash
./merge-dev-to-main.sh
```

## Если изменения не видны

1. Проверьте Vercel Dashboard → последний деплой из `main`
2. Убедитесь, что статус "Ready"
3. Очистите кеш браузера/Telegram
4. Перезапустите бота (если он запущен на сервере)
