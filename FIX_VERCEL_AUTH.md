# Исправление проблемы с аутентификацией Vercel

## Проблема
При открытии мини-приложения через Telegram бот открывается страница входа в Vercel вместо самого приложения.

## Причина
Vercel по умолчанию может требовать аутентификацию для Preview deployments, особенно если проект настроен как приватный.

## Решение

### Вариант 1: Отключить требование аутентификации в Vercel Dashboard (рекомендуется)

1. Зайдите в **Vercel Dashboard** → ваш проект `step-one-app`
2. Перейдите в **Settings** → **General**
3. Найдите раздел **"Deployment Protection"** или **"Preview Deployments"**
4. Убедитесь, что **"Require Authentication"** или **"Password Protection"** **ОТКЛЮЧЕН** для Preview deployments
5. Сохраните изменения

### Вариант 2: Проверить настройки проекта

1. В Vercel Dashboard → **Settings** → **General**
2. Проверьте:
   - **Visibility**: должен быть "Public" (не "Private")
   - **Password Protection**: должен быть отключен
   - **Deployment Protection**: должен быть отключен для Preview

### Вариант 3: Использовать Production deployment вместо Preview

Если Preview требует аутентификацию, можно использовать Production URL:
- Обновите `MINIAPP_BASE_URL` в боте на production URL (после деплоя в main)

### Вариант 4: Настроить Custom Domain

Если у вас есть домен, можно настроить его в Vercel:
1. Settings → **Domains**
2. Добавьте свой домен
3. Используйте этот домен в боте

## Проверка

После изменений:
1. Подождите несколько минут (Vercel может кешировать настройки)
2. Попробуйте открыть URL напрямую в браузере (не через Telegram)
3. Если открывается приложение без логина - проблема решена
4. Перезапустите бота на сервере, чтобы он использовал обновленный URL

## Альтернативное решение

Если ничего не помогает, можно:
1. Задеплоить изменения в `main` ветку
2. Использовать production URL вместо Preview
3. Или настроить стабильный Preview URL через Vercel CLI
